# 工作流限制（Workflow Rules）

此文档用于限制 GitHub Copilot Agent 的工作流行为。\
**Agent 必须在执行任何操作前加载并遵守以下规则。**

------------------------------------------------------------------------

# 代码风格（Code Style Rules）

## 1. 通用（General）

1.  所有代码文件缩进必须使用 **2 个空格**，禁止使用 Tab 字符。
2.  Agent 自动生成、修改或补全的代码都必须遵守本规范。

------------------------------------------------------------------------

## 2. C# 风格（C# Style）

1.  `.cs` 文件的 namespace 必须使用 **文件作用域命名空间**格式：

    **允许：**

    ``` csharp
    namespace MyProject.Services;
    ```

    **禁止：**

    ``` csharp
    namespace MyProject.Services
    {
    }
    ```

2.  禁止将 `if` 判断写成单行语句。

    **允许：**

    ``` csharp
    if (value > 0)
    {
        Handle();
    }
    ```

    **禁止：**

    ``` csharp
    if (value > 0) Handle();
    ```

------------------------------------------------------------------------

# 说明文档（Documentation Rules）

1.  Agent 必须在执行任何任务前**阅读并理解项目根目录的 `Overview.md`**。
2.  当 Agent 对项目代码或结构进行任何修改时，必须同步更新项目根目录的
    **`Overview.md`**，确保文档始终反映当前状态。
3.  新增或修改功能时，必须尽可能编写对应的
    **单元测试**，并提升整体代码覆盖率。
